# DeepSeek Opus 蒸馏实验报告 — Round 1 (Choice-Head → SFT)

## 实验概述

| 项目 | 内容 |
|------|------|
| **实验名称** | DeepSeek Opus R1: choice-head 蒸馏 → GT SFT |
| **Run ID** | 20260409_103609_deepseek_opus_r1 |
| **日期** | 2026-04-09 |
| **基线** | 77.11% (纯 GT SFT, seeded) |
| **DeepSeek 旧方案最佳** | 79.52% (两阶段选择性蒸馏, 不可复现, stability mean=73.49%) |
| **核心创新** | 首次将 choice-head 方法应用于 DeepSeek 教师 |
| **关键差异** | DeepSeek 有 14.14% 的 teacher-GT 不一致率 (vs 豆包 0.33%) |

## 方案

### 策略：三步流水线

1. **预处理 — 软标签生成**
   - DeepSeek 原始数据只有硬标签 (TeacherAnswer)，无概率分布
   - 通过 label smoothing (ε=0.25) 将硬标签转换为软分布
   - 例：TeacherAnswer=A → {A: 0.80, B: 0.05, C: 0.05, D: 0.05, E: 0.05}

2. **Stage 1 — Choice-Head 蒸馏** (1 epoch)
   - 仅在选项 token 位置 (A-E) 计算损失
   - Loss = α × KL(student, teacher_soft) + (1-α) × CE(student, GT)
   - 672/672 样本全部可用（vs 豆包 604/672）

3. **Stage 2 — GT 生成式 SFT** (2 epochs)
   - 从 Stage1 的 LoRA 权重继续训练
   - Loss = 纯 CE (α=0.0)

### 教师数据

- 来源：DeepSeek V3 API 推理结果
- 原始文件：`teacher_train_selective.jsonl` (来自 deepseek_v3_selective 实验)
- 软标签转换：`prepare_soft_labels.py` (label smoothing ε=0.25)
- 全量匹配：672/672 (100%)
- **教师不一致率：14.14% (95/672 样本)** — 这是与豆包的关键区别
- 平滑后熵均值：0.7777，峰值均值：0.80

## 分析

1. **81.93% 是所有实验的历史最高值**，超越了之前的 80.72% (+1.21pp)
2. **DeepSeek 的"新信息"是双刃剑**：14% 不一致率带来更高峰值 (81.93%)，但也导致更大方差 (std=3.09 vs 豆包 2.21)
3. **均值低于基线**：mean=76.87% < 77.11%，蒸馏在平均意义上有害
4. **方差主导结果**：最好 81.93% 和最差 72.29% 相差 9.64pp，比任何超参差异都大
5. **与豆包的关键差异**：豆包教师太"安全"（0.3% 不一致），产生保守结果；DeepSeek 教师更"冒险"（14% 不一致），产生极端结果

## 下一步建议

1. **多模型集成**：选取 DeepSeek (seed=11) + 豆包 (seed=42/55) 的模型做 majority vote
2. **增加训练数据**：672 样本是根本瓶颈，远比蒸馏方法重要
3. **探索 α 对不一致样本的差异化处理**：对 14% mismatch 样本使用更低的 α
4. **不建议继续纯调参**：当前方法论的天花板已经很清楚
